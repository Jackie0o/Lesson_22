Основи синтаксису JAVASCRIPT. Структура коду. Інструкції. Режим use strict. Уроки JAVASCRIPT з нуля.

// Варіант №1
console.log('Вчимо JS')
Відповідь: Помилки в браузері не буде. Але краще добавляти ";" в кінці інструкції, так як може виникнути ситуація, що людини продовже писати код та в майбутньому при стисканні коду може виникнути помилка. Краще думати на перед.
// Варіант №2

console.log('JS')

console.log('Вчимо')

Відповідь: Помилки в браузері не буде. Але краще добавляти ";" в кінці інструкції, так як може виникнути ситуація, що в майбутньому при стисканні коду може виникнути помилка. Краще думати на перед.
// Варіант №3

console.log('Вчимо')

console.log('JS')

Відповідь: в браузері видасть помилку. У першої інструкції в кінці нема крапки з комою ";", тобто вона незачинена і зразу починається (на тому же рядку) нова інструкції, що є помилкою. Після кожнох інструкції треба ставити крапку з комою ";".
// Варіант №4

console.log('Вчимо);

console.log('JS');

Відповідь: в браузері видасть помилку. Лапки в першій інструкції повині бути закриті.

Змінні у JAVASCRIPT. Константи. Ключові слова LET CONST VAR. Уроки JAVASCRIPT з нуля 2020

// Варіант №1

let user = 'Вася'

let age = 36

Відповідь: в браузері видасть помилку. У першій об'яві зміної в кінці нема крапки з комою ";", тобто вона незачинена і зразу починається (на тому же рядку) нова об'ява зміної, що є помилкою. Після кожної об'яви зміної треба ставити крапку з комою ";".
// Варіант №2

let myage = 36;

Відповідь: Помилки в браузері не буде. Але це помилка згідно стилю надання імен до змін lowerCamelCase. Правильно myAge.
// Варіант №3

const BLOCK_SIZE = 14 + 50;

Відповідь: Помилки в браузері не буде. Але імена констант, чиї значення відомі заздалегідь, прийнято записувати у верхньому регістрі, а слова розділяти нижнім підкресленням. Значення константи не відомо заздалегідь. Її треба вирохувати.
// Варіант №4

const BLOCK_HEIGHT = 100;

Відповідь: Помилки в браузері не буде. Імена констант, чиї значення відомі заздалегідь, прийнято записувати у верхньому регістрі, а слова розділяти нижнім підкресленням. Значення константи BLOCK_HEIGHT відомо заздалегідь. Все вірно.
// Варіант №5

if (true) {

var size = 15;

}

console.log(size);

Відповідь: В Console видасть число 15. Так змінна "size" об'явленна в блоці if і начебто по правилам області видимості змінних команда "console.log(size)" не має видати значення, але це спеціфіка об'яви змінної через "var" і команда "console.log(size)" видасть значення. Все вірно.

Типи даних JAVASCRIPT. Null String Undefined Boolean Number та інші. Уроки JAVASCRIPT з нуля 2020

// Варіант №1

let userAge = 36 ;

let userInfo = "Фрілансер ${userAge}";

Відповідь: Якщо треба вивести в зміну userInfo значення - Фрілансер ${userAge}, то все вірно. Але, мені здається тут задумка використовувати значення змінної userAge в змінній userInfo, тоді це не вірно. Правильно використовувати зворотні лапки (``). Вірно `Фрілансер ${userAge}`;.
// Варіант №2

let userHeight = 145 / 0;

// Повернеться значення NaN

console.log(userHeight);

Відповідь: В Console видасть значення Infinity. Значення NaN повертається, коли не можливе математичне обчислення. Так в школі нас вчили, що на 0 не можно ділити, але в JS можно =). Це не є помилкою, просто число буде нескінчене, тобто Infinity.
// Варіант №3

let userName;

// Повернеться тип даних Null

console.log(typeof userName);

Відповідь: В Console видасть значення Undefined. Значення Null представляє відсутність будь-якого об'єктного значення. Це значення, яке ми привласнюємо змінній. Якщо змінна об'явленна і до неї не привласнили значення, то тоді тип данних буде Undefined - не визначено.
// Варіант №4

let userSize = "45" / "8";

// Повернеться тип даних Number

console.log(typeof userSize);

Відповідь: Все вірно, в Console видасть значення Number. Так ми бачимо, що строка ділиться на строку. Але оператор "/" переводить строки в цифри (якщо в строках тільки цифри, в іншому випадку буде NaN) і ми отримаємо, що 45 ділиться на 8. Це дорівнює 5.625 , що є числом - Number.
